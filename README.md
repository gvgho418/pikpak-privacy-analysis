### 云端下载：ISP看不到你在存什么

这是PikPak的核心技术亮点，也是隐私保护的关键环节。

PikPak官网说得很清楚：**你的云下载发生在PikPak的远程服务器上，所有服务器通信使用加密协议，ISP无法监控你保存的内容。**

这个逻辑是这样的：你把一个磁力链接或BT种子丢给PikPak，是PikPak的服务器去互联网上抓取这个文件，然后存进你的私人空间。你自己的网络连接只是在和PikPak服务器通信，而且是加密的。所以在ISP那一层看到的，只是你在访问PikPak，而不是你在下载什么具体内容。

相比自己用BT客户端下载，这个方式在本地网络层的暴露面确实小得多。

PikPak的运营主体是**PIKCLOUD PTE. LTD.**，在新加坡注册。这个地理位置很重要——它不受中国内容监管体系的约束，也就意味着PikPak不会像国内网盘那样主动扫描并删除所谓"敏感资源"。

很多用户反映，在国内网盘里根本存不进去的资源，PikPak几秒钟就秒存了。这不是bug，这是它的设计初衷。

## PikPak隐私政策：它实际收集了哪些数据

说完优点，得聊点让人有点不安的地方。

PikPak有一份英文版隐私政策（在官网可以查到），写得相当详细。它明确列出了会收集以下信息：

**账户信息**：注册时你提供的联系地址、电话、邮箱、用户名（如果你用Google登录，就只有Google账户信息）。

**你的内容**：PikPak会访问你上传和存储的文件内容，用于确认其中没有违法内容（色情、侵权、仇恨内容等）。这一点在政策里写得很直白——**它们会看你存了什么**，至少扫描层面是这样。

**设备信息**：IP地址、浏览器类型、设备标识符等，用于反欺诈和故障排查。

**使用信息**：你在账户里的操作记录（分享、编辑、查看、创建文件等）、日志、搜索历史等。

**位置信息**：为了提供更好体验，可能收集地理位置。

说实话，这些收集项目和大多数海外云服务差不多，甚至比某些国内平台公开的要少。但有一点需要注意：**PikPak会扫描你存储的内容，这不是"完全不看"的私密**。

所以更准确的理解是：PikPak的"私密"指的是外部（ISP、第三方）无法轻易窥探你的传输行为，而不是说PikPak自己对你的内容完全视而不见。
